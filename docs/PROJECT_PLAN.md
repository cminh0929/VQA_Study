# VISUAL QUESTION ANSWERING (VQA)

## 1. PROJECT OVERVIEW

### 1.1. Requirements

**Objective**: Use **CNN** and **LSTM** to build architectures for the Visual Question Answering task.

**Input**: 
- Image
- Question

**Output**: 
- Answer — **MUST** be generated by an **LSTM Decoder**

**Architecture Requirements**: Build different architectures based on:
1. **Attention Mechanism**: With vs Without Attention
2. **Training Strategy**: From-scratch vs Pretrained model

**Evaluation Requirements**: 
- Define appropriate metrics
- Compare models using these metrics

### 1.2. Scope & Data

- **Data Source**: MS COCO Dataset
- **Scale**: ~3,000 - 5,000 images containing 10 common animal species
- **Annotation Method**: Use YOLO for object detection, K-Means for color detection, auto-generate Q&A pairs

**Question Types (by priority):**

1. **Animal Recognition** — PRIMARY GOAL
   - Example: "What animal is in the image?" → "dog", "cat", "bird"
   - Suitable for: **ALL 4 models**
   - Expected accuracy: 50-90%

2. **Color Recognition** — PRIMARY GOAL
   - Example: "What color is the dog?" → "black", "white", "brown"
   - Suitable for: **ALL 4 models**
   - Expected accuracy: 40-80%

3. **Yes/No Questions** — SECONDARY GOAL
   - Example: "Is there a cat in the image?" → "yes", "no"
   - Suitable for: **ALL 4 models**
   - Expected accuracy: 60-95%

4. **Simple Counting** —  EXPERIMENTAL
   - Example: "How many dogs are there?" → "1", "2", "3"
   - Suitable for: **Pretrained models ONLY (1, 2)**
   - Expected accuracy: 45-65% (pretrained), ~10-20% (from-scratch)

### 1.3. Model Architecture

#### 1.3.1. Architecture Overview

The VQA system uses a **CNN-LSTM Encoder-Decoder** architecture:

```
┌─────────────────────────────────────────────────────────────┐
│                         INPUT                                │
│  • Image (224x224x3)                                        │
│  • Question (text sequence)                                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    ENCODER STAGE                             │
│                                                              │
│  ┌──────────────────┐         ┌──────────────────┐         │
│  │  CNN Encoder     │         │  LSTM Encoder    │         │
│  │  (Image)         │         │  (Question)      │         │
│  │                  │         │                  │         │
│  │  Pretrained:     │         │  Embedding +     │         │
│  │   ResNet50       │         │  2-layer LSTM    │         │
│  │  From-scratch:   │         │  Hidden: 512-D   │         │
│  │   ScratchCNN     │         │                  │         │
│  │  (ResNet-style)  │         │                  │         │
│  │                  │         │                  │         │
│  │  Output: 2048-D  │         │  Output: 512-D   │         │
│  │  or (49, 2048)   │         │                  │         │
│  └──────────────────┘         └──────────────────┘         │
│           ↓                            ↓                     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    FUSION MODULE                             │
│                                                              │
│  Variant 1: Simple Concatenation (No Attention)            │
│  • Concat(image_feat, question_feat) → Fused features      │
│                                                              │
│  Variant 2: Spatial Attention (7x7 regions)                │
│  • Softmax over 49 spatial regions guided by question      │
│  • Weighted sum → focused image representation             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              DECODER STAGE (LSTM Decoder)                    │
│                                                              │
│  • 2-layer LSTM Decoder (Hidden: 512-D)                    │
│  • Input: Fused features + previous word embedding         │
│  • Teacher Forcing during training                          │
│  • Greedy decoding during inference                         │
│  • Output: Softmax over vocabulary (word-by-word)          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                         OUTPUT                               │
│  Answer sequence (e.g., "dog", "black", "2")               │
└─────────────────────────────────────────────────────────────┘
```

### 1.4. Technical Specifications

- **Framework**: PyTorch
- **Image Encoder**: ResNet50 (2048-D)
- **Question Encoder**: LSTM (2 layers, hidden_size=512)
- **Answer Decoder**: LSTM (2 layers, hidden_size=512)
- **Attention**: Spatial Attention (Bahdanau-style, 7x7=49 regions)
- **Optimizer**: Adam (lr=0.001)
- **Loss Function**: Cross-Entropy Loss
- **Batch Size**: 32
- **Max Sequence Length**: Question: 20, Answer: 10
- **Teacher Forcing**: Scheduled decay from 1.0 → 0.5
- **Gradient Clipping**: max_norm=5.0

### 1.5. Evaluation Metrics

#### 1.5.1. Primary Metrics

**1. Accuracy ⭐ MOST IMPORTANT**
- Exact match accuracy between prediction and ground truth
- Overall accuracy + per-category accuracy

**2. BLEU Score**
- BLEU-1 (unigram): suitable for single-word answers
- BLEU-4 (4-gram): suitable for longer answers

**3. F1 Score**
- Harmonic mean of precision and recall
- Per-category evaluation

#### 1.5.2. Comparison Plan

| Comparison Purpose | Primary Metric | Secondary Metric |
|-------------------|----------------|------------------|
| **Overall Performance** | Overall Accuracy | BLEU-1, F1 Score |
| **Impact of Attention** | Accuracy (Model i vs i+1) | Attention Visualization |
| **Impact of Pretrained** | Accuracy (Pretrained vs Scratch) | Convergence Speed |
